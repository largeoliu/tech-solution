# 双模块日志格式统一方案

## 概述

统一用户模块和订单模块的日志格式，全部采用 JSON 格式输出，便于日志收集和检索。

## 架构设计

### 现状分析
- 用户模块：使用自定义文本格式日志
- 订单模块：使用 log4j 原生格式
- 差异：字段不统一，难以关联查询

### 统一方案
采用 JSON 格式作为统一日志格式，字段包含：timestamp, level, module, traceId, message

## 实现方案

### 任务分解
| 模块 | 任务 |
|------|------|
| 用户模块 | 替换日志框架为 JSONLayout |
| 订单模块 | 配置 JSON 格式化插件 |
| 共用 | 统一 traceId 生成策略 |

## 风险评估

| 风险项 | 影响 | 缓解措施 |
|--------|------|----------|
| 日志量增加 | 低 | JSON 紧凑度优于文本 |
| 兼容性 | 中 | 双写过渡期后下线旧格式 |
