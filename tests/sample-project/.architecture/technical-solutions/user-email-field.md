# 用户模块邮箱字段方案

## 概述

在用户模块增加邮箱字段，用于支持通知发送功能。当前用户模型缺少 email 字段，本方案在 User 实体中新增 email 属性，作为用户身份信息的扩展。

## 架构设计

邮箱字段定位为用户实体的标准属性，承载通知地址信息。

- 在用户实体 (User) 中新增 `email` 字段
- 类型: VARCHAR(255)
- NULL 支持: 是
- 默认值: NULL

## 实现方案

1. 编写数据库迁移脚本，为用户表添加 email 列，默认值为 NULL
2. 若需唯一约束，分两步执行：先添加字段，后添加唯一索引
3. 所有读取邮箱的逻辑必须支持 NULL 处理
4. 存量数据无需清洗

## 风险评估

| 风险项 | 影响 | 缓解措施 |
|--------|------|----------|
| 存量数据 email 为 NULL | 中 | 所有读取路径支持 NULL |
| 用户隐私 | 高 | 加密存储，遵循数据保护法规 |
