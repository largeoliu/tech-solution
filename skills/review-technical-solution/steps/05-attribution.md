# 步骤 5：归因与分级

## 输入
- 代码取证结果（step-4 产物）
- 方案分类（step-2 产物）
- `.architecture/principles.md`
- 状态文件（含 step-4 结果）

## 操作
1. 所有问题归因到五维度之一：需求对齐、架构对齐、代码现状对齐、完整性、可落地性
2. 按严重级别分级：blocker/major/minor/note
3. 分级基于证据和影响，不因人调整

## 完成标准
- 所有问题都已补齐归因维度与严重级别

## 输出
- 更新状态文件 `checkpoints.step-5`
