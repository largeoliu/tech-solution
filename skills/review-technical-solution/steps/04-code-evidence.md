# 步骤 4：代码取证

## 输入
- 相关项目代码
- 核心主张清单（step-3 产物）
- 状态文件（含 step-3 结果）

## 操作
1. 在代码中核验每条主张，尤其要单独核验所有“新增、拆分、合并、迁移、平行建设、职责转移、能力增强”类主张
2. 每条只能落在三种状态之一：已证实、已证伪、待核验
3. 方案声称复用现有能力时必须找到代码证据
4. 凡方案涉及新增、拆分、合并、迁移、平行建设、职责转移或能力增强，必须主动核验是否存在可直接复用或低风险改造的现有候选；即使方案未列候选，也要搜索并记录搜索范围与结果。搜索范围至少覆盖：同职责对象、命名相近对象、共享基础层、直接入口或调度层
5. 对每条上述主张，至少记录：现有资产是否存在、是否支持直接复用、是否支持低风险改造、选定路径理由是否成立
6. 找不到证据时标记为待核验或已证伪

## 完成标准
- 每条主张都已取得证据状态
- 所有“新增、拆分、合并、迁移、平行建设、职责转移、能力增强”类主张都已完成“是否可复用 / 可改造”的反向取证
- 找不到证据时标记为待核验或已证伪

## 输出
- 更新状态文件 `checkpoints.step-4`

## 门控
未完成不得进入第 5 步

相关代码发生变更
