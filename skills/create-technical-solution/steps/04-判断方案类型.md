# 步骤 4：判断方案类型

## 输入
- 方案主题
- 模板结构
- 状态文件

## 操作
1. 按"方案类型速判"判断命中哪一类或哪几类（新功能方案、重构或替换方案、跨系统或平台能力方案、安全/合规/数据保护方案、性能/容量/成本优化方案）
2. 确定必答问题、易漏风险、评审重点
3. 确定推荐参与成员
4. 同步判定流程级别 `flow_tier`：
   - `light`：单模块小改动，不涉及新建核心能力、不涉及跨系统调用链变化、不涉及职责迁移、无高风险兼容性问题
   - `moderate`：不满足 `light`，但仍不涉及新建核心能力、拆分/迁移/平行建设、职责边界重划、多系统集成或高风险兼容性改造
   - `full`：凡涉及新建核心能力、拆分、迁移、平行建设、职责转移、多系统集成或高风险兼容性改造，均判定为 `full`
5. 按 `flow_tier` 预写本次流程的 `required_artifacts`：
   - `light`：至少包含 `WD-CTX`、`WD-SYN-LIGHT`
   - `moderate`：至少包含 `WD-CTX`、`WD-TASK`、`WD-SYN`
   - `full`：至少包含 `WD-CTX`、`WD-TASK`、`WD-EXP-*`、`WD-SYN`
6. 仅当 `flow_tier`、`required_artifacts`、推荐参与成员都已明确时，才允许后续步骤继续

## 完成标准
- 方案主类型已确定
- 参与成员集合已初步确定
- 必答问题、易漏风险、评审重点已记录

## 输出
- 更新状态文件 checkpoints.step-4

## 门控
方案类型未确定时不能进入第 5 步。

## 回退信号
方案类型判断本身发生变化。
