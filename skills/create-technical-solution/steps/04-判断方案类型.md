# 步骤 4：判断方案类型

## 输入
- 方案主题
- 模板结构
- 状态文件

## 操作
1. **【强制】前置条件自检**：
   - 运行 `python /path/to/run-step.py --state <状态文件路径>` 查看当前门禁与修复建议
   - 确认步骤 3 已完成（template_loaded 为 true、working_draft_path 已创建）
   - 若验证失败，按 `repair_plan[]` 修复后重试
   - 展示自检结果（通过/不通过 + 具体原因）
2. 按"方案类型速判"判断命中哪一类或哪几类（新功能方案、重构或替换方案、跨系统或平台能力方案、安全/合规/数据保护方案，性能/容量/成本优化方案）
3. 确定必答问题、易漏风险、评审重点
4. 确定推荐参与成员
5. 所有流程统一为 full 级别，中间产物固定为 `WD-CTX`、`WD-TASK`、`WD-EXP-*`、`WD-SYN`
6. 使用 `python /path/to/run-step.py --state <状态文件路径> --complete --summary "<步骤4摘要>" --solution-type "<方案类型>"` 原子写入：
   - `checkpoints.step-4.solution_type`
   - `required_artifacts`（固定为 `["WD-CTX", "WD-TASK", "WD-EXP-*", "WD-SYN"]`）
   严禁先推进步骤，再手工补改状态文件

## 完成标准
- 方案主类型已确定
- 参与成员集合已初步确定
- 必答问题、易漏风险、评审重点已记录

## 输出
- 更新状态文件 checkpoints.step-4

## 门控
方案类型未确定时不能进入第 5 步。

## 回退信号
方案类型判断本身发生变化。
