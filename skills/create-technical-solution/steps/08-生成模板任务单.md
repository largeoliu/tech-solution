# 步骤 8：生成模板任务单

## 输入
- 当前模板
- 共享上下文清单
- 状态文件

## 操作
1. 进入本步骤前先通过 `python /path/to/run-step.py --state <状态文件路径>` 查看门禁。若未通过，优先根据 `repair_plan[]` 修复，并遵循其中 `depends_on_steps` 的前置顺序，再参考 `summary.recommended_repair_sequence`、`summary.recommended_rollback_step`、`summary.missing_artifacts` 与 `issues[*].repair_guidance` 确认修复效果后重试
2. 为当前模板中的**每个真实槽位**生成最小任务单，只保留：
   - `槽位标识`
   - `必须消费的共享上下文`
   - `参与专家`
   - `每位专家必答问题`
   - `建议落位槽位`
   - `落位表达要求`
   - `缺口或阻塞项`
3. `必须消费的共享上下文` 只能引用已存在的 CTX 编号；不得在任务单中重复抄写 CTX 事实详情
4. `每位专家必答问题` 必须具体到当前槽位，作为该槽位分析的唯一罗盘，不能写成泛泛议题；凡槽位涉及新增、拆分、合并、迁移、平行建设、职责转移或能力增强，问题必须至少覆盖：
   - 当前目标能力
   - 现有候选资产
   - `复用 / 改造 / 新建` 三种路径比较
   - 放弃复用/改造的证据
   - 现有模板槽位落位要求
5. 若同一槽位涉及多个受影响能力单元，任务单必须逐个能力单元列出问题，不得合并成泛化问题
6. `禁止越界项` 不再作为独立字段重复保存，但其约束仍然有效：未比较现有候选前不得直接推荐新建；不得仅用空泛理由支持新建；不得预设模板中存在固定复用评估小节、表格或列名
7. 严格按当前模板顺序列出槽位；任务单只能服务当前模板槽位，不得额外引入模板外组织层或隐含章节；不得退化成“背景/总体/详细设计/测试/上线”等粗粒度大章节映射
8. 将 `WD-TASK` 区块写入 working draft 后，由 `run-step.py` 同步 `produced_artifacts`
9. 仅当 `WD-TASK` 已写入时，才把状态文件 `can_enter_step_9` 和 `can_enter_step_10` 置为 true
10. 更新 `checkpoints.step-8` 时，`summary` 只能写短格式流程摘要，例如：`完成；写入 WD-TASK；slots=19；gate: step-10 ready`
11. 向用户展示摘要

## 完成标准
- 当前模板每个槽位都有对应任务单条目
- `WD-TASK` 已覆盖当前模板的全部槽位，不存在漏槽位或合并槽位
- 所有最小必填字段完整
- `必须消费的共享上下文` 只引用已存在的 CTX 编号
- `每位专家必答问题` 足够具体，可直接驱动专家逐槽位分析
- 涉及新增、拆分、合并、迁移、平行建设、职责转移或能力增强的槽位，`每位专家必答问题` 已按能力单元逐条覆盖：
  - 现有候选资产比较
  - 新建必要性判断
  - 现有模板槽位落位要求
- 任务单仅围绕当前模板推进，无模板外组织层
- WD-TASK 区块已写入 working draft
- 摘要已向用户展示

## 输出
- 更新状态文件 checkpoints.step-8
- WD-TASK 写入 working draft

## 门控
以下情况不能组织专家分析，不得进入第 9 步：
- 模板任务单未完成
- `WD-TASK` 未覆盖当前模板全部真实槽位
- 存在无效 CTX 引用
- 专家必答问题仍停留在泛化描述
- 涉及新增、拆分、合并、迁移、平行建设、职责转移或能力增强的槽位未按能力单元纳入现有候选比较问题或现有模板槽位落位要求

## 回退信号
模板槽位定义、槽位边界、参与专家分配、阻塞条件变化。
