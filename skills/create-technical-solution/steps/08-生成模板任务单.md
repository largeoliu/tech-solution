# 步骤 8：生成模板任务单

## 输入
- 当前模板
- 共享上下文清单
- 状态文件

## 操作
1. 为每个槽位明确：槽位标识、槽位语义、可填写边界、必须消费的共享上下文、参与专家、每位专家必答问题、禁止越界项、依赖槽位、建议落位槽位、落位表达要求、缺失即停止的上下文、缺口或阻塞项
2. `必须消费的共享上下文` 只能引用已存在的 CTX 编号；`每位专家必答问题` 必须具体到当前槽位，作为该专家分析的唯一罗盘，不能写成泛泛议题
3. 凡槽位涉及新增、拆分、合并、迁移、平行建设、职责转移或能力增强，`每位专家必答问题` 必须至少覆盖：当前目标能力是什么、现有候选资产有哪些、`复用 / 改造 / 新建` 三种路径如何比较、若不选复用或改造证据是什么、若选择新建现有对象为何不能安全承载、这些结论最终应落入当前模板哪个已有槽位以及用何种表达粒度承载
4. 若同一槽位涉及多个受影响能力单元，任务单必须逐个能力单元列出问题，不得合并成泛化问题
5. 对上述槽位，`禁止越界项` 必须明确禁止：未比较现有候选前直接推荐新建；仅用“更清晰”“更优雅”“更容易扩展”等空泛理由支持新建；预设模板中存在固定的复用评估小节、表格或列名
6. 严格按当前模板顺序列出槽位；任务单只能服务当前模板槽位，不得额外引入模板外组织层或隐含章节
7. 将 WD-TASK 区块写入 working draft
8. 向用户展示摘要

## 完成标准
- 当前模板每个槽位都有对应任务单条目
- 所有必填字段完整
- `必须消费的共享上下文` 只引用已存在的 CTX 编号
- `每位专家必答问题` 足够具体，可直接驱动专家逐槽位分析
- 涉及新增、拆分、合并、迁移、平行建设、职责转移或能力增强的槽位，`每位专家必答问题` 已按能力单元逐条覆盖现有候选资产比较、新建必要性判断，以及现有模板槽位落位要求
- 任务单仅围绕当前模板推进，无模板外组织层
- WD-TASK 区块已写入 working draft
- 摘要已向用户展示

## 输出
- 更新状态文件 checkpoints.step-8
- WD-TASK 写入 working draft

## 门控
模板任务单未完成，存在无效 CTX 引用，专家必答问题仍停留在泛化描述，或涉及新增、拆分、合并、迁移、平行建设、职责转移或能力增强的槽位未按能力单元纳入现有候选比较问题 / 现有模板槽位落位要求时，不能组织专家分析，不得进入第 9 步。

## 回退信号
模板槽位定义、槽位边界、参与专家分配、阻塞条件变化。
