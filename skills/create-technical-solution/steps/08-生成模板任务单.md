# 步骤 8：生成模板任务单

## 输入
- 当前模板
- 共享上下文清单
- 状态文件

## 操作
1. 为每个槽位明确：槽位标识、槽位语义、可填写边界、必须消费的共享上下文、参与专家、每位专家必答问题、禁止越界项、依赖槽位、缺失即停止的上下文、缺口或阻塞项
2. `必须消费的共享上下文` 只能引用已存在的 CTX 编号；`每位专家必答问题` 必须具体到当前槽位，作为该专家分析的唯一罗盘，不能写成泛泛议题
3. 严格按当前模板顺序列出槽位；任务单只能服务当前模板槽位，不得额外引入模板外组织层或隐含章节
4. 将 WD-TASK 区块写入 working draft
5. 向用户展示摘要

## 完成标准
- 当前模板每个槽位都有对应任务单条目
- 所有必填字段完整
- `必须消费的共享上下文` 只引用已存在的 CTX 编号
- `每位专家必答问题` 足够具体，可直接驱动专家逐槽位分析
- 任务单仅围绕当前模板推进，无模板外组织层
- WD-TASK 区块已写入 working draft
- 摘要已向用户展示

## 输出
- 更新状态文件 checkpoints.step-8
- WD-TASK 写入 working draft

## 门控
模板任务单未完成，存在无效 CTX 引用，或专家必答问题仍停留在泛化描述时，不能组织专家分析，不得进入第 9 步。

## 回退信号
模板槽位定义、槽位边界、参与专家分配、阻塞条件变化。
