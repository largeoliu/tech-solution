# 步骤 1：定题与范围判断

## 输入
- 用户提供的方案主题或需求描述
- 状态文件

## 操作
1. **【强制】前置条件自检**：
   - 运行 `python /path/to/run-step.py --state <状态文件路径>` 查看当前门禁与修复建议
   - 仅当 `run-step.py` 内部 validator 通过并写入 receipt 后，才允许更新步骤 1 状态
2. 提取方案主题、问题、目标、非目标、约束、影响范围
3. 模糊项若会影响主题、目标、非目标或影响范围判断，且无法从现有上下文推断时，按 AskUserQuestion 四步法向用户澄清：
   - **Re-ground**：说明当前正在为「[方案主题]」定题与界定范围（1-2 句）
   - **Simplify**：用 16 岁能懂的语言解释为什么需要澄清该项，不用内部术语
   - **RECOMMENDATION**：给出推荐选项及理由，附 `Completeness: X/10`
   - **Options**：A) ... B) ... C) ...
   其余模糊项记录假设并继续
4. 生成 ASCII kebab-case 的短横线文件名（slug）
5. 运行 `python /path/to/run-step.py --state <状态文件路径> --complete --summary "完成；slug=<slug>；paths=1；gate: step-2 ready" --slug <slug>`
6. 输出方案主题一句话摘要

## 完成标准
- 主题一句话已明确并写入摘要
- 问题、目标、非目标有记录
- 约束与依赖已识别
- 影响范围已界定
- ASCII kebab-case slug 已生成

## 输出
- 更新状态文件 checkpoints.step-1
- 更新 `final_document_path`

## 门控
主题、目标、非目标、约束、影响范围未明确时停在第 1 步，不得进入第 2 步。

## 回退信号
用户修正问题定义、目标、非目标或影响范围。
