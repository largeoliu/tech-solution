# 步骤 7：构建共享上下文

## 输入
- `.architecture/principles.md`
- 代码与配置
- 现有实现
- 相关文档
- repowiki（如存在）
- 状态文件

## 操作
1. 整合所有来源形成共享上下文
2. 每条上下文包含：上下文编号（CTX-01 递增）、来源、结论或约束、适用槽位、可信度或缺口
3. 若 repowiki 存在，先读取目录、导航与总览，再补充与当前方案直接相关的内容，并记录已覆盖范围与仍待补证范围
4. 将 WD-CTX 区块写入 working draft
5. 向用户展示摘要

## 完成标准
- 每条上下文都有完整字段
- 关键槽位上下文已齐全或识别为阻塞项
- 如存在 repowiki，已记录相关范围覆盖情况或待补证范围
- WD-CTX 区块已写入 working draft
- 摘要已向用户展示

## 输出
- 更新状态文件 checkpoints.step-7
- WD-CTX 写入 working draft

## 门控
关键槽位无 CTX 支撑时停止，不得进入第 8 步。

## 回退信号
共享事实、原则解释、现状判断、关键约束变化。
