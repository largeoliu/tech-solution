# 步骤 2：检查 .architecture 前置文件

## 输入
- 状态文件

## 操作
1. **【强制】前置条件自检**：
   - 运行 `python /path/to/run-step.py --state <状态文件路径>` 查看当前门禁与修复建议
   - 确认步骤 1 已完成（本轮方案文件名已确定，后续路径可落盘）
   - 若验证失败，按 `repair_plan[]` 修复后重试
   - 展示自检结果（通过/不通过 + 具体原因）
2. 确认 `.architecture/members.yml` 存在且可读
3. 确认 `.architecture/principles.md` 存在且可读
4. 确认 `.architecture/templates/technical-solution-template.md` 存在且可读
5. 任一缺失时停在第 2 步并给出缺失清单，引导用户使用 `bootstrap-architecture`
6. 将 `checkpoints.step-2` 写成结构化对象，至少包含：
   - `summary`
   - `prerequisites_checked: true`
   - `missing_files: []`（如有缺失则列出）
7. 更新本步骤时只能通过 `run-step.py --complete` 完成，不得直接手改 state

## 完成标准
- 三文件存在且可读
- 任一缺失时停在第 2 步并给出缺失清单

## 输出
- 更新状态文件 checkpoints.step-2

## 门控
三文件任一缺失时立即停止，不得进入第 3 步。

## 回退信号
前置文件被删除或移动。
