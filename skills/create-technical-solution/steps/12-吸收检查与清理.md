# 步骤 12：吸收检查与清理

## 输入
- 最终文档
- working draft
- 状态文件

## 操作
1. 先执行流程完整性断言：运行 `python /path/to/run-step.py --state <状态文件路径>` 查看门禁。此校验只负责确认：
   - `can_enter_step_12 = true`
   - `final_document_path` 已写入且最终文档存在
   - 最终文档槽位顺序与当前模板一致
   若未通过，必须保留 working draft 和状态文件，并优先根据 `repair_plan[]` 修复，且遵循其中 `depends_on_steps` 的前置顺序，并按 `completion_checks` 确认每项修复已完成；若还需补充判断，再参考 `summary.recommended_repair_sequence`、`summary.recommended_rollback_step`、`summary.missing_artifacts` 与 `issues[*].repair_guidance` 后重试；通过后再进入清理
   - `checkpoints.step-12.summary` 已写入且非空
2. 执行吸收检查自检清单：
   - 步骤连续性：`completed_steps` 无跳号、无越级完成，`current_step` 与当前阶段一致
   - 结构对齐：一级章节数量与顺序与模板一致，无新增模板未定义章节
   - Working draft 完整性：全流程只有 1 份 working draft，slug 与最终技术方案文件一致，下游内容仅消费已写入的 `WD-*` 稳定区块
   - 上下文追溯：每个 CTX 条目来源可追溯到文件、目录、配置项或用户输入；无法映射到槽位的 CTX 已记录原因
   - 任务单消费：若 `flow_tier` 为 `moderate` 或 `full`，每个槽位必须消费的 CTX 编号均已存在且被核销，专家必答问题具体，无模板外组织层，无基于"缺失即停止"上下文却未标记阻塞；`light` 流程跳过任务单时必须有显式跳过记录
   - 专家证据：若 `flow_tier` 为 `full`，每个 `WD-EXP-*` 参与槽位都包含 `参与槽位`、`决策类型`、`核心理由`、`关键证据引用`、`未决点`；凡 `新建` 结论均在 `关键证据引用` 中覆盖不可复用与不可改造证据
     - 如存在 repowiki，相关槽位已纳入且未替代 CTX grounding
     - `light` / `moderate` 流程跳过专家产物时必须有显式跳过记录
   - 收敛闭合：
     - `light` 流程的 `WD-SYN-LIGHT`，以及 `moderate` / `full` 流程的 `WD-SYN`，都必须保留选定路径、关键证据与建议落位
     - 凡 `新建` 结论均在 `关键证据引用` 中覆盖不可复用、不可改造与新建必要性的证据
     - 进入最终文档的槽位都能稳定映射回模板，缺 CTX、冲突未解或 `模板承载缺口` 未解槽位已阻塞
   - 失效标记：每个 `WD-IMPACT-*` 先声明失效范围，再记录保持有效内容；无可复用内容时明确写 `无`，失效结论已标注作废
   - 禁止项检查：最终文档无模板禁止的结构变化
   - 成稿一致性：所有必需的 WD-* 区块有实际内容，最终文档只吸收未作废且已完成收敛的结论
   - 现有资产决策闭合：凡涉及新增、拆分、合并、迁移、平行建设、职责转移或能力增强的能力单元，最终文档均已在当前模板已有槽位中写入现有候选、选定路径及支撑该路径成立的关键证据；若为 `新建`，其关键证据已覆盖不可复用、不可改造与新建必要性说明；不存在依赖新增模板结构承载这些结论的情况
3. 仅在吸收检查通过后，使用 `python /path/to/run-step.py --state <状态文件路径> --complete --summary "<步骤12摘要>"` 原子执行以下动作：
   - 写入短格式 `checkpoints.step-12.summary`，例如：`完成；validator_passed=true；deleted=2`
   - 写入 `checkpoints.step-12.validator_passed = true`
   - 设置 `absorption_check_passed: true`、`cleanup_allowed: true`
   - 删除 working draft 和状态文件
   - 验证删除成功后直接结束流程
   步骤 12 开始前不得提前设置这两个字段；若当前状态里这两个字段已提前为 true，必须先回退为 false 再继续
4. 未通过时必须同时保留 working draft 和状态文件，并继续修正；不得删除任何中间产物

## 完成标准
- 吸收检查已通过（全部打勾）
- `completed_steps` 连续且与 `current_step` 一致，不存在跳步完成
- working draft 和状态文件已删除（通过时）或均保留并修正（未通过时）

## 输出
- 删除 `.architecture/.state/create-technical-solution/[slug].yaml`
- 删除 working draft（通过时）

## 门控
未完成第 12 步不得结束流程；删除状态文件后不得再继续调用 `run-step.py --complete` 推进旧状态。

## 回退信号
无（最终清理步骤）。
