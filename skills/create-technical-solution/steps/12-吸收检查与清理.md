# 步骤 12：吸收检查与清理

## 输入
- 最终文档
- working draft
- 状态文件

## 操作
1. 执行吸收检查自检清单（6 项）：
   - 结构对齐：一级章节数量与顺序与模板一致，无新增模板未定义章节
   - 上下文追溯：每个槽位结论能引用 CTX 编号，每个 WD-EXP-* 建议已进入收敛纪要或记录未采纳理由
   - 收敛闭合：每个槽位分歧点都有选定写法，被否决方向未删除
   - 任务单消费：每个槽位必须消费的 CTX 编号已核销，无基于"缺失即停止"上下文却未标记阻塞
   - 禁止项检查：最终文档无模板禁止的结构变化
   - Working draft 完整性：所有 WD-* 区块有实际内容，最终文档与 working draft 一致
2. 通过后删除 working draft 和状态文件
3. 未通过时保留并修正

## 完成标准
- 吸收检查已通过（全部打勾）
- working draft 已删除（通过时）或已修正（未通过时）

## 输出
- 删除 `.architecture/.state/[slug].yaml`
- 删除 working draft（通过时）

## 门控
未完成第 12 步不得结束流程。

## 回退信号
无（最终清理步骤）。
