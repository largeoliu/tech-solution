# 步骤 12：吸收检查与清理

## 输入
- 最终文档
- working draft
- 状态文件

## 操作
1. 执行吸收检查自检清单（10 项）：
   - 结构对齐：一级章节数量与顺序与模板一致，无新增模板未定义章节
   - Working draft 完整性：全流程只有 1 份 working draft，slug 与最终技术方案文件一致，下游内容仅消费已写入的 `WD-*` 稳定区块
   - 上下文追溯：每个 CTX 条目来源可追溯到文件、目录、配置项或用户输入；无法映射到槽位的 CTX 已记录原因
   - 任务单消费：每个槽位必须消费的 CTX 编号均已存在且被核销，专家必答问题具体，无模板外组织层，无基于"缺失即停止"上下文却未标记阻塞
   - 专家证据：每个 `WD-EXP-*` 参与槽位都包含 `支撑理由`、`现有候选`、`决策类型`；凡 `新建` 结论均保留 `不可复用证据` 与 `不可改造证据`；如存在 repowiki，相关槽位已纳入且未替代 CTX grounding
   - 收敛闭合：每个槽位分歧点都有选定写法，被否决方向未删除；`WD-SYN` 已保留 `候选方案对比`、`选定路径`、`建议落位槽位`、`落位写法约束`，且凡 `新建` 结论均保留 `不可复用证据`、`不可改造证据`、`新建必要性证据`；进入最终文档的槽位都能稳定映射回模板，缺 CTX、冲突未解或 `模板承载缺口` 未解槽位已阻塞
   - 失效标记：每个 `WD-IMPACT-*` 先声明失效范围，再记录保持有效内容；无可复用内容时明确写 `无`，失效结论已标注作废
   - 禁止项检查：最终文档无模板禁止的结构变化
   - 成稿一致性：所有 WD-* 区块有实际内容，最终文档只吸收未作废且已完成收敛的结论
   - 现有资产决策闭合：凡涉及新增、拆分、合并、迁移、平行建设、职责转移或能力增强的能力单元，最终文档均已在当前模板已有槽位中写入现有候选、选定路径及支撑该路径成立的关键证据；若为 `新建`，已写入不可复用证据、不可改造证据、新建必要性证据；不存在依赖新增模板结构承载这些结论的情况
2. 仅在吸收检查通过后删除 working draft 和状态文件
3. 未通过时同时保留 working draft 和状态文件，并继续修正

## 完成标准
- 吸收检查已通过（全部打勾）
- working draft 和状态文件已删除（通过时）或均保留并修正（未通过时）
