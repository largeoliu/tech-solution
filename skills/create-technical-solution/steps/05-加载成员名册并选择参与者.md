# 步骤 5：加载成员名册并选择参与者

## 输入
- `.architecture/members.yml`
- 状态文件
- 方案类型

## 操作
1. 读取 `.architecture/members.yml`
2. 默认至少包含系统架构师
3. 根据方案类型与名册中的自定义专家决定最终参与成员集合
4. 向用户展示成员名单并确认

## 完成标准
- `.architecture/members.yml` 已读取
- 最终参与成员名单已确定（含选定理由）
- 用户已知悉且未提出更换

## 输出
- 更新状态文件 checkpoints.step-5

## 门控
参与成员未确定时不能构建共享上下文，不得进入第 6 步。

## 回退信号
只影响参与成员集合本身。
