# 步骤 10：按模板逐槽位协作收敛

## 输入
- 所有专家产物
- 共享上下文
- working draft
- 状态文件

## 操作
1. 仅基于已写入的 WD-EXP-* 与 CTX，按模板槽位收敛成：目标能力、现有候选、共同结论、分歧点、候选方案对比、选定路径（`复用` / `改造` / `新建`）、选定写法、原则冲突与取舍、不可复用证据、不可改造证据、新建必要性证据、兼容性影响、建议落位槽位、落位写法约束、模板承载缺口、未决问题；只保留能够稳定映射回当前模板槽位的结论
2. `候选方案对比` 至少覆盖三类路径：直接复用现有资产、在现有资产上改造、新建资产；若某一类路径不适用，必须写明不适用原因与证据
3. 记录已核销的共享上下文、上下文冲突处理、未采纳理由、仍缺共享上下文、仍待决策；若关键 CTX 仍缺失、冲突未解、`新建` 路径缺少证明前两类路径不可接受的证据，或现有模板槽位无法承载必要结论且会导致结构越界，必须将该槽位标记为阻塞，不得进入最终成稿
4. 将 WD-SYN 区块写入 working draft，不得以最终文档替代 WD-SYN
5. 向用户展示摘要

## 完成标准
- 每个槽位都有收敛纪要
- 候选方案对比已覆盖，且已包含 `复用 / 改造 / 新建` 最低比较集
- 进入最终成稿的槽位都能稳定映射回模板已有槽位
- 上下文冲突已处理，或已显式标记为阻塞槽位
- 关键 CTX 缺失的槽位已从最终成稿候选中剔除并记录原因
- 所有 `新建` 结论都已附带 `不可复用证据`、`不可改造证据`、`新建必要性证据` 与兼容性说明
- 每个受影响能力都已给出 `建议落位槽位` 与 `落位写法约束`；若现有模板无法承载，已显式记录 `模板承载缺口`
- WD-SYN 区块已写入 working draft
- 摘要已向用户展示

## 输出
- 更新状态文件 checkpoints.step-10
- WD-SYN 写入 working draft

## 门控
协作收敛未完成，WD-SYN 未落盘，仍存在未标记阻塞的缺 CTX / 冲突槽位，任一 `新建` 结论缺少前两类路径不可接受的证据，或任一槽位存在未解决的 `模板承载缺口` 时，不能进入最终成稿，不得进入第 11 步。

## 回退信号
某些槽位的共同结论、分歧点、选定写法、未采纳理由变化。
