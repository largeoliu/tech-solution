# 步骤 10：按模板逐槽位协作收敛

## 输入
- 所有专家产物
- 共享上下文
- working draft
- 状态文件

## 操作
1. 进入本步骤前先核验状态文件门禁：
   - `light`：`can_enter_step_10` 为 true，且 `produced_artifacts` 已包含 `WD-CTX`
   - `moderate`：`can_enter_step_10` 为 true，且 `produced_artifacts` 已包含 `WD-CTX`、`WD-TASK`
   - `full`：`can_enter_step_10` 为 true，`produced_artifacts` 已包含 `WD-CTX`、`WD-TASK`、`WD-EXP`，`selected_members` 已落状态，且专家产物集合必须覆盖这些成员
   任一条件不满足即阻塞，不得收敛
2. 按 `flow_tier` 组织收敛，并按槽位逐个落盘，而不是等待所有槽位统一汇总后一次性写入：
   - `light`：输出紧凑版 `WD-SYN-LIGHT`，每个槽位至少包含 `目标能力`、`候选路径对比`、`选定路径`、`关键证据`、`建议落位槽位`、`未决问题或阻塞`
   - `moderate` / `full`：仅基于当前槽位已写入的 `WD-EXP-*`（`moderate` 可为空）与相关 CTX，收敛成最小闭合集：`目标能力`、`候选方案对比`、`选定路径`、`选定写法`、`关键证据引用`、`建议落位槽位`、`模板承载缺口`、`未决问题`
3. `候选方案对比` 至少覆盖三类路径：直接复用现有资产、在现有资产上改造、新建资产；若某一类路径不适用，必须写明不适用原因与证据
4. 记录当前槽位已核销的共享上下文、上下文冲突处理、未采纳理由、仍缺共享上下文、仍待决策；若关键 CTX 仍缺失、冲突未解、`新建` 路径缺少证明前两类路径不可接受的证据，或现有模板槽位无法承载必要结论且会导致结构越界，必须将该槽位标记为阻塞，不得进入最终成稿
5. 将 `WD-SYN` 或 `WD-SYN-LIGHT` 区块按槽位增量写入 working draft，不得以最终文档替代收敛区块；并把对应产物追加到状态文件 `produced_artifacts`
6. 仅当所有非阻塞槽位都已写入本步骤所需收敛产物，且无未标记阻塞槽位时，才把状态文件 `can_enter_step_11` 置为 true
7. 向用户展示摘要

## 完成标准
- 每个槽位都有收敛纪要
- 候选方案对比已覆盖，且已包含 `复用 / 改造 / 新建` 最低比较集
- 进入最终成稿的槽位都能稳定映射回模板已有槽位
- 上下文冲突已处理，或已显式标记为阻塞槽位
- 关键 CTX 缺失的槽位已从最终成稿候选中剔除并记录原因
- 所有 `新建` 结论都已在 `关键证据引用` 中覆盖不可复用、不可改造与新建必要性的证据说明
- 每个受影响能力都已给出 `建议落位槽位`；若现有模板无法承载，已显式记录 `模板承载缺口`
- `light` 流程已写入 `WD-SYN-LIGHT`；`moderate` / `full` 流程已写入 `WD-SYN`
- 摘要已向用户展示

## 输出
- 更新状态文件 checkpoints.step-10
- `WD-SYN` 或 `WD-SYN-LIGHT` 写入 working draft

## 门控
协作收敛未完成、所需收敛区块未落盘、仍存在未标记阻塞的缺 CTX / 冲突槽位，任一 `new-build` 结论缺少前两类路径不可接受的证据，或任一槽位存在未解决的 `模板承载缺口` 时，不能进入最终成稿，不得进入第 11 步。

## 回退信号
某些槽位的共同结论、分歧点、选定写法、未采纳理由变化。
