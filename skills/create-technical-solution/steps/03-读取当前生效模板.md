# 步骤 3：读取当前生效模板

## 输入
- `.architecture/templates/technical-solution-template.md`
- 状态文件

## 操作
1. **【强制】前置条件自检**：
   - 运行 `python scripts/validate-state.py --state <状态文件路径> --step 3 --flow-tier <flow_tier> --format json`
   - 确认步骤 2 已完成（prerequisites_checked 为 true）
   - 若验证失败，按 `repair_plan[]` 修复后重试
   - 展示自检结果（通过/不通过 + 具体原因）
2. 读取模板标题、章节层级、说明文字、占位说明
3. 沿用步骤 1 的 slug；若缺失则补齐 ASCII kebab-case slug
4. 预留 working draft 路径：`.architecture/technical-solutions/working-drafts/[slug].working.md`
5. 记录模板结构快照（提取所有一级章节，按顺序编号为 SLOT-01, SLOT-02, ...）
6. 将模板结构快照写入 working draft 头部
7. 将 `template_loaded: true` 和 `working_draft_path` 写入状态文件
8. 展示模板结构摘要并自动继续

## 完成标准
- 模板标题、一级章节数量和顺序已记录
- 每个一级章节语义已明确
- working draft 路径已创建并写入模板结构快照
- 模板结构摘要已展示

## 输出
- 更新状态文件 checkpoints.step-3
- working draft 头部写入模板结构快照

## 门控
模板结构摘要未稳定时不能判断方案类型，不得进入第 4 步。

## 回退信号
当前生效模板发生变化。
