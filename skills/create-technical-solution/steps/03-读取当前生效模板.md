# 步骤 3：读取当前生效模板

## 输入
- `.architecture/templates/technical-solution-template.md`
- 状态文件

## 操作
1. 读取模板标题、章节层级、说明文字、占位说明
2. 生成短横线文件名
3. 预留 working draft 路径：`.architecture/technical-solutions/working-drafts/[slug].working.md`
4. 记录模板结构快照（提取所有一级章节，按顺序编号为 SLOT-01, SLOT-02, ...）
5. 将模板结构快照写入 working draft 头部
6. 向用户确认模板结构

## 完成标准
- 模板标题、一级章节数量和顺序已记录
- 每个一级章节语义已确认
- working draft 路径已创建并写入模板结构快照
- 模板结构已向用户确认

## 输出
- 更新状态文件 checkpoints.step-3
- working draft 头部写入模板结构快照

## 门控
模板结构摘要未稳定时不能判断方案类型，不得进入第 4 步。

## 回退信号
当前生效模板发生变化。
