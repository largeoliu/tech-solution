# 技术方案阶段性播报协议

这个文档只定义“何时展示什么、何时失效重进”。它负责对话内的过程可见性契约，不改动正式技术方案文档的模板结构，也不引入新的侧车产物。

## 1. 目标与边界

### 目标

- 在正式技术方案成稿前，按阶段向用户展示稳定、可解释、可归因的中间产物。
- 让用户尽快看到 `专家产物` 和 `协作收敛纪要`，降低长时间等待的黑盒感。
- 允许用户在阶段边界介入，并从最近受影响的阶段边界恢复，而不是整条链路全部重来。

### 边界

- 中间产物默认只在当前对话中展示，不新增正式文档或侧车文件。
- 正式技术方案仍以当前模板为唯一正文骨架。
- 不展示 scratchpad、原始推理链路、未成形草稿或无法稳定归因的内容。
- 讨论过程只展示结构化结果，不展示原始争论聊天记录。

## 2. 阶段边界

阶段性播报只在以下阶段边界触发：

1. `共享上下文清单阶段`：共享上下文来源、适用槽位和关键缺口稳定后，先展示 `共享上下文清单`，再允许进入 `模板任务单阶段`。
2. `模板任务单阶段`：模板槽位、参与专家、必答问题和阻塞条件稳定后，才允许进入后续分析。
3. `专家按模板逐槽位分析阶段`：单个专家补齐当前模板槽位所需必填字段后，立即展示该专家的 `专家产物`。
4. `按模板逐槽位协作收敛阶段`：全部专家输入完成后，统一展示一份 `协作收敛纪要`。
5. `严格模板成稿阶段`：用户看过收敛结果后，再把选定方向、权衡、风险和未决问题按当前模板严格成稿。

阶段边界要求：

- 未达到当前阶段的稳定边界前，不提前展示半成品。
- 一旦用户修改前提、目标、约束或影响范围，先判断最近受影响的阶段边界，再决定是否重进。
- 重进时只能复用仍然有效的稳定结论，失效内容必须显式作废。

### 共享上下文清单：展示契约

`共享上下文清单` 是第 6 步共享证据的对话内展示包装，不是另一套字段 schema。字段级定义、必填项和内容约束由 `references/solution-process.md` 中的 `共享上下文清单` 约束；本文件只定义用户可见形态和展示时机。

共享上下文来源、约束、适用槽位和缺口稳定后，必须先展示 `共享上下文清单`，再展示 `模板任务单`，才允许进入 `专家按模板逐槽位分析阶段`。

对用户展示时，至少应包含：

- 明确标题：`共享上下文清单`
- 结构化主体：按 `references/solution-process.md` 的共享上下文字段顺序展示
- 必要说明：若存在低可信度、冲突上下文或关键缺口，必须显式保留

最小展示外壳如下：

```markdown
## 共享上下文清单

[按 references/solution-process.md 的共享上下文清单格式渲染正文]
```

使用规则：

- 必填字段未补齐时，不展示该 `共享上下文清单`。
- 展示顺序应与 canonical 共享上下文字段顺序一致，不在本文件重复定义字段细节。
- 长内容采用“摘要优先，详细展开”的表达方式；先给稳定判断，再补充细节。

## 3. 模板任务单

### 模板任务单：展示契约

`模板任务单` 是对 `references/solution-process.md` 中 `模板任务单` 的对话内展示包装，不是另一套字段 schema。它必须被视为 `references/solution-process.md` 中模板任务单字段 schema 的对话渲染。字段级定义、必填项和内容约束以 `references/solution-process.md` 为准；本文件只定义用户可见形态和展示时机。

模板槽位、参与专家、必答问题和阻塞条件稳定后，必须先展示 `模板任务单`，再进入 `专家按模板逐槽位分析阶段`。

对用户展示时，至少应包含：

- 明确标题：`模板任务单`
- 结构化主体：按 `references/solution-process.md` 的模板任务单字段顺序展示
- 上下文绑定：`本槽位必须消费的共享上下文` 与 `缺失即停止的上下文` 必须显式保留
- 必要说明：若存在阻塞槽位、不适用专家或依赖缺口，必须显式保留

最小展示外壳如下：

```markdown
## 模板任务单

[按 references/solution-process.md 的模板任务单格式渲染正文]
```

使用规则：

- 必填字段未补齐时，不展示该模板任务单。
- 按 `references/solution-process.md` 的 canonical `模板任务单` schema 与当前模板槽位顺序稳定展示，不得自行换序或删减必填字段。
- 长内容采用“摘要优先，详细展开”的表达方式；先给稳定判断，再补充细节。
- 模板发生变化或槽位边界失效时，已有 `模板任务单` 必须作废，并至少回退到 `模板任务单阶段` 重进。

## 4. 专家产物

### 专家产物：展示契约

`专家产物` 是对 `references/solution-process.md` 中 `专家按模板逐槽位分析` 的对话内展示包装，不是另一套字段 schema。它必须被视为 `references/solution-process.md` 中模板逐槽位字段 schema 的对话渲染。字段级定义、必填项和内容约束以 `references/solution-process.md` 为准；本文件只定义用户可见形态和展示时机。

对用户展示时，至少应包含：

- 专家身份：专家名称与角色头衔
- 视角说明：该专家本轮重点关注什么
- 结构化主体：按 `references/solution-process.md` 的模板逐槽位字段顺序展示
- 上下文引用：`已使用的共享上下文编号`、`未使用原因`、`结论是否超出上下文支持`

最小展示外壳如下：

```markdown
### [专家名称] - [角色头衔]

**视角说明**：
[该专家本轮重点关注什么]

[按 references/solution-process.md 的专家按模板逐槽位分析格式渲染正文]
```

使用规则：

- 必填字段未补齐时，不展示该专家产物。
- 展示顺序应与 canonical 字段顺序一致，不在本文件重复定义字段细节。
- 长内容采用“摘要优先，详细展开”的表达方式；先给判断，再补充细节。
- 每张卡片只代表该专家当前阶段的独立判断，不提前替其他专家收敛。
- 全部相关槽位的专家产物稳定后，进入按模板逐槽位协作收敛阶段。

## 5. 协作收敛纪要

`协作收敛纪要` 是 `references/solution-process.md` 中 canonical `按模板逐槽位协作收敛` 结构的对话内展示形态。它必须被视为 `references/solution-process.md` 中模板逐槽位收敛 schema 的对话渲染。两者不是两套不同内容：

- `按模板逐槽位协作收敛`：字段级 schema 与收敛契约，作为 canonical 来源
- `协作收敛纪要`：面向用户展示的渲染结果，内容必须完整映射 canonical `按模板逐槽位协作收敛`

全部专家输入完成后，必须先展示 `协作收敛纪要`，再进入正式成稿。

对用户展示时，至少应包含：

- 明确标题：`协作收敛纪要`
- 收敛主体：按 `references/solution-process.md` 的模板逐槽位收敛字段顺序渲染
- 上下文核销：`本槽位已核销的共享上下文`、`上下文冲突如何处理`、`仍缺哪条共享上下文`
- 必要说明：若存在强分歧、被否决方向或未决问题，必须显式保留

最小展示外壳如下：

```markdown
## 协作收敛纪要

[按 references/solution-process.md 的按模板逐槽位协作收敛格式渲染正文]
```

使用规则：

- `协作收敛纪要` 不得偏离或删改 canonical `按模板逐槽位协作收敛` 的必需内容。
- 必须显式保留争议点、被否决方案和未选原因。
- 不输出原始争论聊天记录，只输出可评审的收敛结果。
- 如果仍有强分歧，纪要中必须保留分歧，不得伪装为已达成一致。
- 若任一槽位缺少已核销的共享上下文，不进入 `严格模板成稿阶段`。

## 6. 严格模板成稿阶段说明

- `协作收敛纪要` 展示完成后，才能进入 `严格模板成稿阶段`。
- `严格模板成稿阶段` 只负责把已稳定的结论映射回当前模板，不发明模板外可见结构。
- 若模板无法安全承载必需信息，应停止并向用户确认，而不是绕过模板约束继续写。
- 中间展示产物不是正式交付物，最终以 `.architecture/technical-solutions/` 中落盘文档为准。

## 7. 展示边界规则

展示边界规则如下：

1. 只展示稳定、可解释、可归因的结果。
2. 不展示 scratchpad、原始推理链路、未成形草稿或内部自检痕迹。
3. 先展示摘要，再按需要补充细节，避免把长篇原始内容直接倾倒给用户。
4. 不把尚未完成收敛的猜测表述为当前结论。
5. 用户修改前提后，先说明失效范围，再进入重算或重进。

## 8. 用户介入、失效与阶段重进

### 用户中途介入

- 用户可在任一已展示阶段后补充约束、修正目标、缩小范围或提出异议。
- 收到新信息后，先判断它影响了哪些既有结论，再确定最近受影响的阶段边界。

### 失效处理

- 已失效的 `专家产物` 或 `协作收敛纪要` 不能继续表述为当前有效结论。
- 失效说明必须明确：什么发生了变化、哪些结论受影响、哪些结论仍然有效。
- 模板发生变化时，至少回退到 `模板任务单`。
- 如果变化影响候选方案比较或选定方向，必须至少回到 `按模板逐槽位协作收敛阶段`；如果变化影响专家基础判断，必须回到相应 `专家按模板逐槽位分析阶段`。

### 阶段重进

- 重进从最近受影响的阶段边界开始，不无故回退到更早阶段。
- 未受影响且仍然成立的稳定结论可以保留，但必须与失效内容清楚分隔。
- 重进完成后，应重新展示新的专家产物或新的协作收敛纪要，再进入 `严格模板成稿阶段`。

## 9. 变更影响说明

当用户新增信息导致重算、回退或阶段重进时，使用下面模板：

```markdown
## 变更影响说明

- 触发变更：[用户新增或修正的信息]
- 受影响内容：[哪些模板任务单槽位、哪些专家槽位分析、哪些协作收敛结论、哪些模板映射内容失效]
- 最近受影响的阶段边界：[模板任务单阶段 / 专家按模板逐槽位分析阶段 / 按模板逐槽位协作收敛阶段 / 严格模板成稿阶段]
- 保持有效的内容：[仍然成立、可直接复用的稳定结论]
- 下一步动作：[从哪个阶段重进，接下来会先展示什么]
```

要求：

- 如果已展示的 `模板任务单` 内容受影响，必须明确说明其哪些槽位已作废或失效，不得继续视为当前有效版本。
- `最近受影响的阶段边界` 必须明确写出，不得用“需要重新看一下”之类的模糊表述代替。
- `变更影响说明` 先于重进动作输出，避免用户无法判断为何回退。
- 如果无可复用内容，也要明确写“保持有效的内容：无”。
