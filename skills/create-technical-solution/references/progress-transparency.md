# 技术方案阶段展示补充示例

这个文档只补充对话内展示和重进说明的话术示例。字段要求、展示顺序和回退规则仍以 `skills/create-technical-solution/SKILL.md` 为准。

## 1. 展示边界提醒

- 只展示稳定、可解释、可归因的结果。
- 先展示摘要，再按需要补充细节。
- 不展示 scratchpad、原始推理链路、未成形草稿或原始争论记录。
- 不把尚未完成收敛的猜测写成当前结论。

## 2. 各阶段摘要外壳示例

### 共享上下文清单

```markdown
## 共享上下文清单

- 已确认来源：[数量]
- 关键约束：[摘要]
- 关键缺口：[摘要]

详见 working draft：WD-CTX（共享上下文清单）
```

### 模板任务单

```markdown
## 模板任务单

- 已拆分槽位：[数量]
- 阻塞槽位：[若无则写无]
- 需要重点关注的专家分工：[摘要]

详见 working draft：WD-TASK（模板任务单）
```

### 专家产物

```markdown
### [专家名称] - [角色头衔]

- 本轮重点槽位：[摘要]
- 关键判断：[摘要]
- 待补证项：[若无则写无]

详见 working draft：WD-EXP-[expert-slug]（专家产物）
```

### 协作收敛纪要

```markdown
## 协作收敛纪要

- 已收敛槽位：[数量]
- 仍有分歧的槽位：[若无则写无]
- 推荐进入成稿的方向：[摘要]

详见 working draft：WD-SYN（协作收敛纪要）
```

## 3. 重进说明示例

当用户新增约束、修正目标或调整范围后，可按下面结构说明：

```markdown
## 变更影响说明

- 触发变更：[摘要]
- 已失效内容：[摘要]
- 最近受影响的阶段边界：[第 N 步]
- 仍然有效的内容：[若无则写无]
- 下一步动作：[从哪一步重进，以及会先展示什么]

详见 working draft：WD-IMPACT-[n]（变更影响说明）
```

## 4. 常见展示错误

- 还没写入 `WD-*` 区块就先对外展示。
- 直接倾倒完整长文，没有先给用户稳定摘要。
- 重进时没有先说明失效范围。
- 把内部推理过程、争论记录或未成形草稿直接展示给用户。
