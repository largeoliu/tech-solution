# 技术方案模板适配规则

这个文档只定义“如何在当前生效模板的已有章节内组织和落位内容”。它不定义模板结构本身，不负责成员分析，也不修改用户自定义模板的定制流程。这里的“当前生效模板”可能是默认模板，也可能是用户替换后的自定义模板。

## 基本原则

- 当前 `.architecture/templates/technical-solution-template.md` 是唯一正文骨架来源。
- 当前模板不仅是最终正文骨架，也是整条分析链唯一的信息架构。
- 当前生效模板可能是默认模板，也可能是用户替换后的自定义模板。
- 先理解当前模板的语义和层级，再决定内容落位。
- 不允许在任何阶段引入模板外的章节层级。
- 允许在现有章节内部增加更细的小节、列表标题或表格行。
- 不允许新增用户模板没有定义的一级章节。
- 不允许把当前模板重写回默认模板结构。
- 不允许要求用户补标记、占位符、前置元数据或额外契约文件。
- 如果某个必须要写的内容在当前模板内没有合法落点，应停止并向用户确认，而不是自行新增模板外章节。

## 固定落位顺序

### 1. 读取当前模板结构

先读取当前模板的：

- 文档标题
- 一级、二级、三级标题
- 每个标题下已有的说明文字、占位说明和表格结构

如果模板没有明显章节边界，也没有足够的说明文字帮助判断语义，后续落位应更加保守。

### 2. 建立模板语义视图

不要按章节名完全相等匹配，而要按语义理解现有章节是做什么的。判断依据包括：

- 标题文字
- 标题下的说明语句
- 占位内容在要求填写什么
- 已有表格列名或列表提示

### 3. 把收敛后的内容落到现有章节

对协作收敛后需要写回文档的每一项结论，按以下优先级选择当前模板内的归宿：

1. 语义最贴近的现有章节
2. 能与相邻内容共同承载的现有大章节
3. 当前模板中已经用于承载类似信息的表格或列表区域

如果一个现有章节需要承载多项收敛结论：

- 优先在该章节内部增加下一层级的小节
- 如果继续加标题会让结构显得生硬，就改用加粗标签、列表小标题或表格行
- 保持原模板的主层级不变

### 4. 处理合并章节和自由结构

如果模板把多项独立意图合并在一个大章节中：

- 在该大章节内部拆出更细的小节或标签
- 保证信息块内容彼此可辨认
- 不要因为内容多就升级成新的一级章节

如果模板是比较自由的长文结构：

- 优先利用现有段落主题和已有小标题
- 只在当前语境已经明显允许的地方增加下一级小标题
- 若模板整体偏叙述式，不强行套回默认模板布局

### 5. 判断是否需要停止确认

遇到以下情况时，不要猜测，直接停止并向用户确认：

- 同一项结论有多个语义同样接近、但落位后会改变文档重心的候选章节
- 当前模板缺少承载某项关键结论的合理位置
- 把内容塞进最近章节会明显破坏模板意图或阅读结构
- 模板几乎没有结构，无法安全判断应该在哪里承载关键内容

停止时要说明：

- 哪项关键结论无法安全落位
- 当前有哪些可能归宿
- 为什么继续自动放置风险高

## 允许的最小结构变化

- 在现有章节下增加二级或三级小节
- 在现有章节内增加加粗标签
- 在现有列表下补充列表项
- 在现有表格里增加行，必要时增加与当前语义一致的列

## 禁止的结构变化

- 新增用户模板没有定义的一级章节
- 在任何阶段引入模板外的章节层级
- 删除或重排用户模板已有的一级章节
- 把自定义模板改写回默认模板结构
- 要求用户为了配合技能而修改模板格式
- 在模板第一个章节之前插入任何内容（如"概述"、"摘要"、"方案总览"、"执行摘要"等模板中不存在的前置模块）
- 在模板最后一个章节之后追加模板中不存在的一级章节

## 输出自检

完成落位后，至少检查以下几点：

- 所有已收敛内容都已在当前模板的已有章节中找到归宿
- 最终文档的第一个一级章节必须是当前模板的第一个一级章节，之前没有任何额外插入的模块
- 最终文档的最后一个一级章节必须是当前模板的最后一个一级章节，之后没有任何额外追加的一级章节
- 最终文档的一级章节数量和顺序必须与当前模板完全一致
- 当前模板的一级结构保持不变
- 新增的小节或标签没有破坏原模板语气和阅读顺序
- 没有出现回退到默认模板章节名的痕迹
- 自定义模板的原始意图仍然清晰可读
