# 技术方案产出补充示例

这个文档只提供补充示例，不新增主路径规则。执行顺序、必填字段、阶段回退和模板落位规则一律以 `skills/create-technical-solution/SKILL.md` 为准。

## 1. 适合查这个文档的场景

- 主技能已经足够执行，但你想看更长的产物骨架示例。
- 你担心阶段产物写得过短，想核对字段展开方式。
- 你想快速自检哪些地方最容易漏写。

## 2. 共享上下文清单示例骨架

```markdown
## 共享上下文清单

- `上下文编号`：CTX-01
- `来源`：`.architecture/principles.md` / `src/...` / 外部约束文档
- `结论或约束`：
- `适用槽位`：
- `可信度或缺口`：已验证 / 待补证 / 存在冲突
```

补充提醒：

- `来源` 要能让后续成员复查，不要写成“常识”或“经验判断”。
- `适用槽位` 可以多个，但必须和当前模板槽位一致。

## 3. 模板任务单示例骨架

```markdown
## 模板任务单

### [槽位标题]
- `槽位标识`：
- `槽位语义`：
- `可填写边界`：
- `本槽位必须消费的共享上下文`：
- `参与专家`：
- `每位专家必答问题`：
  - [专家 A]
  - [专家 B]
- `禁止越界项`：
- `依赖槽位`：
- `缺失即停止的上下文`：
- `缺口或阻塞项`：
```

补充提醒：

- `每位专家必答问题` 要能直接驱动逐槽位分析，不要写成泛泛的讨论议题。
- `禁止越界项` 要点名最容易被误放到别的槽位的内容。

## 4. 专家产物示例骨架

```markdown
### [专家名称] - [角色头衔]

#### [槽位标识] [槽位标题]
- `是否参与该槽位`：
- `建议写法或建议内容`：
- `支撑理由`：
- `已使用的共享上下文编号`：
- `未使用原因`：
- `结论是否超出上下文支持`：
- `与其他槽位的依赖或冲突`：
- `仍需补证的信息`：
```

补充提醒：

- `建议写法或建议内容` 必须能直接回填到当前模板槽位。
- `结论是否超出上下文支持` 不是装饰字段；不是“没有超出”时就不能直接进入收敛定稿。

## 5. 协作收敛纪要示例骨架

```markdown
## 协作收敛纪要

### [槽位标识] [槽位标题]
- `共同结论`：
- `分歧点`：
- `选定写法`：
- `本槽位已核销的共享上下文`：
- `上下文冲突如何处理`：
- `未采纳理由`：
- `仍缺哪条共享上下文`：
- `仍待决策`：
```

补充提醒：

- 被否决方向和 `未采纳理由` 要保留，方便后续评审和回退。
- `选定写法` 应该已经足够稳定，能直接映射到当前模板。

## 6. 变更影响说明示例骨架

```markdown
## 变更影响说明

- `触发变更`：
- `受影响内容`：
- `最近受影响的阶段边界`：
- `保持有效的内容`：
- `下一步动作`：
```

补充提醒：

- `受影响内容` 要点名具体槽位、具体专家产物或具体收敛结论，不要只写“部分内容失效”。
- `下一步动作` 要明确写从哪一步重进，以及接下来会重新展示什么。

## 7. 常见漏项清单

- 共享上下文没有标 `适用槽位`，导致后续无法核销。
- 模板任务单没有写 `缺失即停止的上下文`，导致阻塞条件不清楚。
- 专家产物没有说明 `未使用原因`，导致无法判断任务单是否被完整消费。
- 协作收敛纪要省略了被否决方向和 `未采纳理由`。
- 变更发生后直接重进，没有先写 `变更影响说明`。
- 最终文档已经落盘，但内容仍无法回溯到 `WD-*` 稳定区块。
