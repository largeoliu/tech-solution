# 需求文档：按标签比例抽样复核

## 背景

当前系统仅支持固定数量抽样，无法按标签维度配置比例抽样。运营团队希望对高风险标签按比例提高复核覆盖率，同时保持旧策略兼容。

## 目标

- 支持按标签配置抽样比例
- 支持与现有按数量模式共存
- 复用现有任务执行链路，避免平行建设
- 明确测试、灰度、回滚方案

## 非目标

- 不新增独立后台系统
- 不改模板结构

## 约束

- 最终技术方案必须遵循当前 `.architecture/templates/technical-solution-template.md`
- 需要说明现有能力复用 / 改造 / 新建的选择依据
- 需要保留可观察性与发布顺序
