# T06-full-服务拆分

## 用例信息
- **ID**: T06
- **场景**: 拆分服务：通知服务拆为消息网关+推送
- **flow_tier**: full
- **集合**: validation

## Query
将单体通知服务拆分为消息网关和推送服务两个独立服务，消息网关负责消息路由和队列，推送服务负责实际发送

## 预期行为 (4 条)
1. flow_tier 判定为 full
2. 拆分方案中选定路径附带职责边界证据和兼容性证据
3. 决策类型为新建的条目同时回答不可复用和不可改造
4. completed_steps 无跳号为 1 到 12
