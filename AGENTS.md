# AGENTS.md

## 1. 适用范围

- 适用于：新建、重写或系统性更新 `skills/<skill-name>/`。
- 不适用于：一次性复盘、项目背景说明、临时协作约定、Skill 使用说明。
- 能稳定脚本化的规则，优先下沉到校验器；本文只保留人工需要判断的部分。

## 2. 必须满足的约束

- 一 Skill 一核心职责；`description` 必须能区分它与相邻 Skill 的边界和触发场景。
- `SKILL.md` 必须独立可执行。读完后，另一个 Agent 应能知道这个 Skill 做什么、如何进入主路径、什么时候不能继续。
- 以下内容必须直接出现在 `SKILL.md`，不能只放在 `references/`：
  - 职责边界
  - 完成标准
  - 必要输入或前提
  - 主路径，或主路径选择规则
  - 关键约束
- 多步骤 Skill 必须写出高层工作流和关键串行依赖。
- 主文件与支持目录不得互相矛盾；同一问题若重复出现，结论和优先级必须一致。
- 每个文件必须符合单一职责原则，不能承担多个职责。
- 下级文档不能引用上级文档，引用链只能是单向的

## 3. 反模式

- 一个 Skill 同时承担多个核心职责。
- 把主路径、必要输入、停止条件或输出边界藏进 `references/`。
- 主文件、引用文件、模板说明互相矛盾。
